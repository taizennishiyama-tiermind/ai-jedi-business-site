1. ファーストビュー
キャッチコピー
「AIを、組織に実装する。」
サブコピー
AI人材を入れるだけでは、組織は変わりません。
AI JEDAIは、人に加えて再現性のあるスキルセット・プリセット環境（AI Dev OS）・運用をパッケージで納品することで、組織へのAI実装を完成させます。
事業概要
AI JEDAIは、AI活用の導入から内製化までを一気通貫で支援するAI実装パートナーです。研修で型を導入し、受託開発で実プロジェクトに適用し、FDE人材投入で運用・内製化まで完成させる。この3ステップを、共通基盤「AI Dev OS」でつなぎ、人だけでなく「動く環境と型」ごと組織に納品します。

2. 課題提起
こんな状況、心当たりはありませんか？
✗  外部のAIエンジニアに任せたが成果が出ず、
   結局自社エンジニアが対応している

✗  試作・PoCはできるが、本番に上がらない

✗  研修を実施したが、現場では誰も動けていない

✗  特定の担当者に依存し、チームに技術が広がらない
根本原因は「仕組み」がないことです。
AI JEDAIは、人を入れるだけでなく「動く環境と型」をセットで組織に渡すことで、これらを同時に解消します。

3. AI Dev OS ── 他社との決定的な違い
AI JEDAIが納品するもの
多くのAI支援サービスは「人」か「研修コンテンツ」を提供して終わります。しかしそれだけでは、こういうことが起きます。
研修は受けた。しかし終わった翌日、現場には何も変わった環境がない。使える型もない。学んだことを実業務に落とし込む手段がなく、気づけば元の開発スタイルに戻っている。
外部エンジニアを入れた。しかし動いているのはその人だけで、チームには何も移転されない。担当者が抜けた瞬間、開発が止まる。
AI JEDAIは、この構造ごと変えます。
AI Dev OSを核に、3つの軸が導入→適用→運用の順で機能します。
導入（研修）
カリキュラムを教えて終わりにしません。研修の成果物としてAI Dev OSを丸ごと納品します。チームは翌日から実際の開発環境と型を使って動き始められる。学んだことが実業務に直結します。
適用（受託開発）
モックを作って渡して終わりにしません。AI駆動でスピードと確度を最大化しながら、ガバナンス・運用設計まで含めて定着させます。研修で導入した型を実際のプロジェクトに適用し、PoC止まりにならない基盤を作ります。
運用（FDE投入）
エンジニアを1名送り込むのではなく、FDE＋AI Dev OS＋運用プレイブックをユニットとして現場に投入します。FDEがプロジェクトを前に進めながら、チームへAI駆動開発の型をインストールしていく。FDEが抜けた後も同じ品質で開発が回り続ける状態が、組織の中に残ります。
成果を個人の能力に依存させない。仕組みとして組織に再現させる。
これがAI JEDAIの提供するものです。
含まれるもの役割開発スターターリポジトリすぐ動き始められるテンプレ一式AI駆動開発プレイブックAIと人間の役割分担・手順を標準化サブエージェント設計仕様〜リリースまでの型を固定GitHub運用テンプレレビュー・CI・品質ゲートの最小セットガバナンス雛形PoC→本番に上げるチェックリスト
研修・受託開発・FDE投入、どの軸から入っても、このAI Dev OSが組織の中に残ります。

AI駆動開発の型を、チームごとインストールする
座学で終わらせません。研修の成果物としてAI Dev OS一式を納品し、終了後すぐに現場が動き出す状態を作ります。対象はDX部門と事業部の混成チームで、非エンジニアでも参加できます。

実践：AI駆動開発の型・GitHub運用・レビュープロセス

期間：2〜4週間

6. STEP 2｜受託開発［適用］
AI駆動で「触れるモック」を最短納品し、意思決定を加速させる
まずモックで要件の不確実性を潰します。従来の受託開発では数週間〜数ヶ月かかる試作を、AI駆動により最短1〜2週間で納品。意思決定者が投資判断できる状態まで素早く可視化し、確度が上がった状態で本開発へ移行するため、手戻りが起きません。このスピードこそ、AI JEDAIの受託開発の最大の強みです。
提供の流れ：
① AIモックスプリント（最短1〜2週）
         ↓
② 技術方針・次フェーズ見積を提示
         ↓
③ 本開発・段階導入
         ↓
④ ガバナンス・運用設計まで定着
開発後の顧客の姿
触れるモックで社内合意が取れており、本開発への投資判断が完了している。ガバナンスと運用設計が整い、PoC止まりにならない基盤ができている。
得られる成果物
✓ 触れて確認できるUIモックアップ
✓ 業務フロー・簡易評価レポート
✓ 技術方針・本開発ロードマップ
✓ ガバナンス・運用設計ドキュメント

事例①｜苗マネ
業務効率化 農業DX 在庫管理システム

「紙面上で行う雑多なメモでの在庫管理に限界を感じている。計算も狂ってくるから結局管理もずさんに。現場のみんなが迷わず使える、シンプルなアプリが欲しいんです。」

→ 2日後に納品
種まき登録・出荷管理・在庫状況・出荷予定表をひとつのダッシュボードで管理できるモックアップ。現場の誰もがリアルタイムで情報共有できるUIを実現。

事例②｜PaintVision
画像処理 建設DX ARシミュレーション
家の壁色、スマホで着せ替え。

「お客様との色決めに時間がかかりすぎる。その場で『こんな感じです』と見せられるツールがあれば、もっとスムーズになるのに。」

→ 翌日に納品
自宅写真のBEFORE/AFTERをスライダーで比較できるUIを実現。モダン・シック、ナチュラル・ウォームなど複数の配色パターンをワンタップで切り替えられるモックアップを納品。

7. STEP 3｜FDE人材投入・内製化［運用］
人と仕組みをセットで現場に投入し、内製化まで完成させる
FDE（Field Deployment Engineer）は、現場に入り込みプロジェクトを前に進める実装型エンジニアです。AI Dev OSと運用プレイブックをセットで持ち込み、チームの自走を支援します。人売りではなく、仕組みごと投入するのがAI JEDAIの形です。
提供内容：

FDEアサインと現場投入
初週からIssue・PR・レビュー・CIを整備し実装成果を提供
AI駆動作法のチームへのインストール（レビュー会・ペア作業）
KPI共有と内製化までの伴走

形態：月次契約
FDE投入後の顧客の姿
個人のスキルに依存せず、チーム全体でAI駆動開発が回っている。担当者が変わっても品質が落ちない仕組みが組織に定着し、内製化が完了している。
得られる成果物
✓ AI駆動開発が自走するチーム体制
✓ 運用プレイブック（組織に残る再現性ある型）
✓ 内製化完了までのKPIレポート
KPI：リードタイム短縮 ／ PR回転数向上 ／ 手戻り率低下 ／ PoC→本番移行率

8. まとめ
AI JEDAIが提供するのは、サービスの組み合わせではありません。
研修でAI Dev OSを組織に導入し、受託開発で実プロジェクトに適用し、FDE投入で運用・内製化まで完成させる。この一本の導線を通じて、「AIが当たり前に使われている組織」を作ることが、私たちの仕事です。
