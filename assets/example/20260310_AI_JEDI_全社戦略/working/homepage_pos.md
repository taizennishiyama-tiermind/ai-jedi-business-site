## 全体の設計思想

**誰に読ませるか**：エンタープライズ企業のDX/IT部門の意思決定者・推進担当者

**何を感じさせるか**：「このままPoC止まりを繰り返すより、AI JEDAIと組んだ方が早い・確実だ」

**構成の方針**：
- 「課題に共感させる」→「解決策を提示する」→「信頼を作る」→「動かす」の順
- AI Dev OSは"説明する"のではなく"成果物として見せる"位置づけ

---

## セクション構成（ドラフト）

---

### Section 1：Hero（ファーストビュー）

**目的**：0.3秒で「自分ごと」にさせる

**キャッチコピー案（複数）**

```
A. 「AI導入が止まるのは、技術の問題じゃない。」
B. 「研修で終わらせない。PoCを本番に変える。」
C. 「学ばせる、作る、使い続ける。AI実装を一気通貫で。」
D. 「あなたの会社のAI開発が、いつまでもPoCな理由。」
```

**サブコピー案**

```
生成AI導入が「属人化」「PoC止まり」「外注依存」で止まっているなら、
環境ごと整えるアプローチが必要です。
```

**CTA**：まずは話を聞く / 資料を見る

---

### Section 2：課題提起（共感ゾーン）

**目的**：ターゲットの悩みを言語化し「わかってる会社だ」と感じさせる

**見出し案**：「こんな状況、心当たりありませんか？」

**課題リスト（箇条書きで3〜5個）**

```
□ 生成AI研修を受けたが、現場が動いていない
□ PoCを作ったが、本番に移行できていない
□ AIを使える人材を採用・育成しようとしているが、評価軸がない
□ 外部委託したが品質・スピードに不満が残る
□ 社内でAI開発を内製化したいが、型がない
```

**締めの一文**：
```
これらは「AIの問題」ではなく、「環境・型・人材」の問題です。
```

---

### Section 3：解決策＋ポジション

**目的**：AI JEDAIが何者かを一言で伝え、差別化を刻む

**見出し案**：「AI JEDAIが提供するのは、人だけじゃない。」

**本文（短く）**：
```
AI JEDAIは「エンタープライズ向け生成AI実装パートナー」です。

研修・開発・人材の3つを、共通基盤（AI Dev OS）でつないで提供します。
「研修で終わる」「外注したら属人化する」ではなく、
顧客の現場が自走できる状態を納品します。
```

**一気通貫の図（テキスト表現）**：
```
学ばせる（研修）→ 作る（AI駆動開発）→ 使わせる（FDE投入）
         └──────── AI Dev OS（共通基盤）────────┘
```

---

### Section 4：3つのサービス

**目的**：具体的な商品を見せ、「自分たちが使えるもの」だと気づかせる

**見出し案**：「3つのアプローチで、AI実装を完成させる」

---

#### ① AI Dev OS Bootcamp（研修）

**タグライン**：「研修が終わったとき、開発環境が動いている。」

**説明**：
- 対象：DX/IT部門＋事業部の混成チーム
- 期間：2〜4週間
- 成果物：AI Dev OS一式 ＋ モックのプロトタイプ ＋ 運用手順
- ポイント：座学で終わらず、顧客の開発がその日から回り始める"型"を納品する

**アップセル**：研修後にモック・スプリントまたはFDE投入へ

---

#### ② AI Rapid Prototype Sprint（受託開発）

**タグライン**：「1〜2週間で、意思決定を前倒しする。」

**説明**：
- ゴール：触れるモックを納品し、本開発の投資判断を加速する
- 成果物：UI/業務フロー/簡易評価 ＋ 技術方針 ＋ 次フェーズ見積の叩き台
- ポイント：要件の不確実性をモックで潰してから本開発へ。失敗コストを最小化する

---

#### ③ FDE Enablement Unit（人材）

**タグライン**：「人を送るのではなく、仕組みごと送る。」

**説明**：
- 提供物：FDE（現場実装型人材）＋ AI Dev OS ＋ 運用プレイブック
- 初週の標準成果例：開発導線の整備 ＋ 動くもの ＋ チームへの作法インストール
- KPI：リードタイム短縮 / PR回転数 / 手戻り率 / PoC→本番移行率

---

### Section 5：AI Dev OS（差別化アセット）

**目的**：「何が違うのか」の根拠を具体的に見せる

**見出し案**：「AI Dev OSとは、再現性ある開発環境の一式です。」

**本文**：
```
AI JEDAIが全事業で使い回す共通基盤。
研修で「導入」→開発で「適用」→人材投入で「運用」する。
```

**含まれるもの（リスト）**：
```
✓ 開発スターターリポジトリ（プロジェクト構成・Issue/PRテンプレ）
✓ サブエージェント設計（AI役割分担の型）
✓ AI駆動開発プレイブック（仕様→実装→リリースの手順）
✓ GitHub運用テンプレ（CI・ラベル・レビュー規約）
✓ エンタープライズ向けガバナンス雛形（PoC→本番チェックリスト）
```

**差別化の一文**：
```
「環境と型」を納品するから、担当者が変わっても再現できる。
```
